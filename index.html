Executive Summary

Radiant Plumbing Service (T3) operates a multi-branch, multi-business-unit P&L process today managed through a ~30–35 tab Excel workbook. Financial data is pulled via the Velixo plugin from Sage Intacct; operational KPIs originate in Service Titan and already land in Domo, but are then manually exported, manipulated in Excel, and re-uploaded. The result is a slow, error-prone, and brittle budgeting & forecasting cycle that demands heavy manual effort from the Finance team and the CIO.

Domo will deliver a production-grade, automated budgeting and forecasting platform that replaces the Excel workbook's core mechanics — revenue build, direct labor model, COGS/OpEx calculations, branch consolidation, and executive-ready outputs — with integrated pipelines, governed inputs, and scenario-ready visualizations. Finance retains full ownership of the forecasting process with faster, more defensible numbers.

Current State Assessment

Process Overview

The current 3+9 Forecast and Annual Budget are authored in a single master Excel workbook comprising approximately 30–35 tabs organized around a branch / location / department hierarchy. Consolidation is performed right-to-left across tabs into a Cons roll-up.

Data Sources In Use Today

  • Sage Intacct (GL Details) — partially in Domo; Department, Location, and Chart of Accounts hierarchies not yet mapped.
  • Sage Intacct via Velixo plugin — pulls actuals into Excel tab-by-tab.
  • Service Titan via Domo connector — automated, reliable; all revenue-build KPIs available.
  • Manual inputs — Marketing %, HR/IT overheads, inflation factors, GM assumptions.

Key Pain Points Identified

  • Excel formula corruption during the live walkthrough — symptomatic of model fragility.
  • Manual export/upload of Domo KPIs into Excel and back into Domo.
  • No self-service for branch GMs — reports are distributed as static Excel files.
  • Sage Intacct is the publishing target for the finished budget — no governed round-trip today.
  • Monthly performance pack and board deck rely on downstream copy/paste.

Model Architecture — Key Components

Revenue Build (per branch × business unit)
Driven by Total Calls Ran, flowing through cancellation rate → demand/maintenance split → conversion (kick-up) rate → average sale → gross sales → discount → net revenue. Color-coded input cells are the only GM-editable drivers. Variants exist for HVAC Install/Service, Plumbing (main line vs. branch line), and related units.
Direct Labor Model
Separate Excel file; maps each employee to COGS or OpEx. Feeds both department tabs and the Admin tab. Identified as second-highest priority for scope discovery before build.
COGS — Inventory & Consumables
Driven by % of revenue using prior-year benchmarks with tunable overrides for cost-savings initiatives or expected operational changes.
OpEx Categories
A blend of (a) % of net revenue (marketing, warehouse supplies, vehicles), (b) prior-year plus inflation (tax/audit, contracts), and (c) stakeholder-provided build files (IT from CIO; HR/uniforms/recruiting from VP HR).
Admin & PE Add-Backs
Centralized on T3 Admin tab; budgeted on prior-year baseline plus planned initiatives.
Consolidation & Output
Right-to-left roll-up into Cons tab → uploaded back to Sage Intacct. GM copies distributed as static Excel tabs by branch. Executives review via Sage P&L reports and downstream monthly packs.

Future State Architecture

Domo becomes the system of record for the budgeting & forecasting workflow. Inputs are captured through governed Domo apps; calculations run deterministically in Magic ETL / Beast Mode; outputs feed both back into Sage Intacct and into executive / GM-facing dashboards.

Solution Pillars

1. Governed Data Foundation

Extend existing Sage integration (GL Details) with Chart of Accounts, Departments, and Locations. Certify Service Titan pipelines already in Domo. Historical backfill to Jan 2024.

2. Input Capture App

Domo custom app with role-based inputs: GM call/conversion/ASP assumptions, Finance overrides, VP HR/IT inputs, marketing %. Replaces color-coded Excel cells with governed forms.

3. Calculation Engine

Magic ETL + SQL dataflows replicating revenue build, labor model, COGS %, OpEx logic, and consolidation — versioned, auditable, scenario-ready.

4. Output & Distribution

Structured export for Sage Intacct upload, GM-facing branch dashboards (Phase 2), monthly financial performance pack (Phase 2), board deck data automation (Phase 3).

Design principle. All Excel-embedded formulas and lookup logic will be replicated in Domo dataflows — not by preserving Excel as a calculation layer. Velixo may be retained temporarily as a parallel-run validation tool, but is not assumed to be part of the final solution.

Reference Data Flow

  • Sage Intacct → Domo (GL Details + COA + Departments + Locations) → Semantic P&L model
  • Service Titan → Domo (existing, automated) → Revenue build inputs
  • Labor data (Payroll system / Excel) → Domo → Direct labor model
  • Input App (GM, Finance, VP HR, CIO) → Writeback dataset → Calculation engine
  • Calculation engine → Budget/Forecast output dataset → (a) Sage upload file, (b) Domo dashboards
